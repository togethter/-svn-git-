创建共享版本库:{
共享版本库的集中情况：
	1搭建服务器，在服务器段设置共享版本库
	2把共享版本库创建在某一个文件夹中
	3把共享版本库创建在u盘中(在网上找资料试一下)
	4把共享版本库托管到第三方的网站上

共享版本库和本地的版本库是不一样的 （git init）
每一个开发者机器上都有一个服务器的数据库
共享版本库的集中情况
1)搭建服务器，在服务器端设置共享版本库(正规公司)
2)把共享版本库创建在某一个文件夹中
3)把共享版本创建在某一个优盘里
4)把共享版本库托管在第三方的网站上面
 
01 创建共享版本库(首先在本地创建一个文件夹里面包含了两个文件夹，一个共享版本库(里面创建一个文件夹XML 在该路径下创建一个共享版本库 cd ....; git --bare init 创建好共享版本库，在项目经理的文件夹下把共享版本库的下载下来 git clone 路径(共享版本库的路径)，之后添加忽略文件 touch .gitignore，在添加忽略的内容该内容从github里搜 gitignore 找到objective的复制黏贴到刚才创建的文件下，之后将忽略文件.gitignore 添加到暂缓区，提交到本地仓库 commit -m，在提交到共享仓库 push)，一个项目经理的本地仓库（manager）)
git --bare init
02 项目经理要对共享版本库做初始化操作
[1]项目经理线连接到共享版本库，把版本库下载到本地 git clone 路径
[2]项目经理完成忽略操作
1)创建忽略文件.gitignore
touch .gitignore
2)设置该文件的内容搜索gitHub gitignore找到objective-c
3)把该文件添加到暂缓区中
git add .gitignore
4)把暂缓区中的内容提交到本地的版本库
git commit -m "创建忽略文件"
5)把本地的版本库提交到共享版本库
git push
}


共享版本库项目初始化
1)通过XCode代码,改代码内部默认会自动做一次添加操作
2)通过终端来把当前的项目提交到本地版本库中
git commit -m "初始化项目"
